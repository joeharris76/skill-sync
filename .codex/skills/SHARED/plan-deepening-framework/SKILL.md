# Plan Deepening Framework

Use before committing to a plan or interpretation.

1. **Obvious answer:** state the straightforward solution/finding first.
2. **What am I missing?** After findings, ask what class of issue the framework misses, what a domain expert would notice, and what production-use assumption is hidden. Route L2 through SHARED/review-protocol/SKILL.md when reviewing; in generative actions apply the question inline (no capture).
3. **Right question?** Before commitment, ask whether the stated problem is the real constraint or an upstream symptom. Document any reframe.
